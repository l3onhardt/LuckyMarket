**日期**: 2026-06-15

## 项目概述

LuckyMarket 是一个公司内部的预测市场平台，允许员工使用虚拟点数参与市场预测交易，同时支持 AI 代理参与。前端需要提供美观、易用的界面，模仿 Polymarket 风格，融合 Apple 设计语言，实现液态玻璃质感。

**核心目标**:
- 美观高端：液态玻璃质感，科技感强
- 极致易用：70 岁老人也能轻松上手
- 模仿 Polymarket：金融感、专业性
- 纯中文界面：无 emoji，使用清晰的图标和文字

## 技术栈

### 前端框架
- **React 18** - 构建用户界面
- **TypeScript** - 类型安全，与后端保持一致

### 其他

## 视觉设计

### 液态玻璃 (Fluid Glass) 设计语言

液态玻璃不是毛玻璃的模糊效果，而是像液体一样流动、透亮、有重量感的玻璃材质。

**核心特征**:
1. **透亮感** - 半透明背景，能看到后面的层次
2. **流动感** - 边缘有微妙的光晕和高光
3. **重量感** - 表面有渐变反射，不是纯色
4. **波动感** - 悬停时像液体波动，不是简单的变色
5. **深度感** - 层叠时有透视和阴影

**CSS 实现**:
```css
/* 液态玻璃卡片 */
.fluid-glass-card {
  background: linear-gradient(135deg,
    rgba(30, 41, 59, 0.7) 0%,
    rgba(51, 65, 85, 0.5) 100%);
  backdrop-filter: blur(40px);
  border: 1px solid rgba(255, 255, 255, 0.08);
  border-radius: 20px;
  box-shadow:
    0 20px 60px rgba(0, 0, 0, 0.3),
    inset 0 1px 0 rgba(255, 255, 255, 0.08),
    inset 0 -1px 0 rgba(0, 0, 0, 0.2);
}

/* 悬停动画 */
.fluid-glass-card {
  transition: all 0.4s cubic-bezier(0.4, 0, 0.2, 1);
}

.fluid-glass-card:hover {
  transform: translateY(-4px) scale(1.01);
  box-shadow: 0 28px 80px rgba(0, 0, 0, 0.4);
  border-color: rgba(255, 255, 255, 0.15);
}
```

### 色彩系统

**克制的三色原则** - 只使用 3 种主色，让界面清晰聚焦：

1. **绿色** (#10b981) - YES / 看涨 / 成功 / 会发生
2. **灰色** (#94a3b8) - NO / 看跌 / 中性 / 不会发生  
3. **蓝色** (#60a5fa) - 操作 / 链接 / 强调 / 选中状态

**背景色系**:
- 主背景: `linear-gradient(180deg, #0f172a 0%, #1e293b 100%)`
- 卡片背景: `linear-gradient(135deg, rgba(30,41,59,0.7), rgba(51,65,85,0.5))`
- 导航背景: `rgba(30, 41, 59, 0.6)` + `backdrop-filter: blur(40px)`

**文字色系**:
- 主标题: #f1f5f9 (接近白色)
- 正文: #cbd5e1 (浅灰)
- 辅助信息: #94a3b8 (中灰)
- 禁用状态: #64748b (深灰)

### 字体层级

**尺寸和权重**:
- 页面标题: 20px, font-weight 700
- 市场标题: 21px, font-weight 600, line-height 1.5
- **概率数字: 48px, font-weight 700** (最显眼)
- 按钮文字: 15px, font-weight 500
- 正文: 15px, font-weight 400
- 辅助信息: 14px, font-weight 400
- 分类标签: 14px, font-weight 500

**最小字号 14px** - 确保老年用户也能清晰阅读。

**字体选择**: Inter (Google Fonts)，备用 -apple-system, system-ui

### 交互设计

**点击区域**:
- 所有按钮最小高度: **48px** (Apple 推荐 44px，我们更大)
- 底部导航按钮宽度: **80px+**, 间距 **32px**
- 市场卡片整个可点击，无需精准点标题
- YES/NO 按钮占据卡片 40% 宽度

**动画原则**:
- 使用 `cubic-bezier(0.4, 0, 0.2, 1)` 缓动曲线，模拟液体波动
- 悬停效果: `translateY(-4px)` + `scale(1.01)`
- 过渡时间: 0.4s (不能太快)
- 页面切换: 淡入淡出 0.3s

**反馈机制**:
- 按钮点击: 颜色变化 + 轻微缩放
- 成功操作: Toast 提示（右上角，3秒自动消失）
- 加载状态: Skeleton 占位 + 微妙的脉动动画
- 错误提示: 红色边框 + 下方文字说明

## 认证和权限

### 认证流程

1. **用户注册**:
   - 输入邀请码、用户名、显示名、密码
   - 后端验证邀请码有效性
   - 创建账户（默认 role=user）
   - 返回 JWT token

2. **用户登录**:
   - 输入用户名、密码
   - 后端验证凭据
   - 返回 JWT token + 用户信息

3. **Token 管理**:
   - Token 存储在 localStorage
   - 每次 API 请求带上 `Authorization: Bearer <token>`
   - Token 过期后跳转到登录页

4. **受保护路由**:
   - 使用 React Router 的路由守卫
   - 未登录用户访问受保护页面 → 重定向到登录页
   - 非管理员访问管理页面 → 显示 403 页面

### 权限控制

**角色定义**:
- `user` - 普通用户（可交易、查看）
- `admin` - 管理员（可创建市场、结算、管理用户）

**权限检查**:
```typescript
// 前端权限检查（UI 层）
const canAccessAdmin = user?.role === 'admin';

// 后端权限检查（API 层）
// 所有管理操作的 API 都需要验证用户角色
```

## 数据流

### API 调用架构

```
React 组件
    ↓
TanStack Query (useQuery / useMutation)
    ↓
API Client (axios + interceptors)
    ↓ 
HTTP Request (带 JWT token)
    ↓
Fastify Backend (localhost:4000)
```

### TanStack Query 使用模式

**查询示例**:
```typescript
// 获取市场列表
const { data, isLoading } = useQuery({
  queryKey: ['markets'],
  queryFn: () => apiClient.get('/markets'),
  refetchInterval: 10000, // 每10秒自动刷新
});
```

**变更示例**:
```typescript
// 提交交易
const tradeMutation = useMutation({
  mutationFn: (params) => apiClient.post(`/markets/${marketId}/trades`, params),
  onSuccess: () => {
    queryClient.invalidateQueries(['markets', marketId]);
    queryClient.invalidateQueries(['portfolio']);
  },
});
```

### 数据库 Schema 调整

**accounts 表需要新增字段**:
```sql
ALTER TABLE accounts ADD COLUMN role TEXT DEFAULT 'user';
ALTER TABLE accounts ADD COLUMN password_hash TEXT;
```

**新增 invite_codes 表**:
```sql
CREATE TABLE invite_codes (
  code TEXT PRIMARY KEY,
  created_by TEXT NOT NULL,
  max_uses INTEGER DEFAULT 1,
  used_count INTEGER DEFAULT 0,
  expires_at TEXT,
  created_at TEXT NOT NULL,
  FOREIGN KEY (created_by) REFERENCES accounts(id)
);
```

## 设计原则总结

1. **少即是多** - 每个页面只做一件事
2. **显而易见** - 无需说明书，看一眼就懂
3. **即时反馈** - 每次操作都有视觉/动画反馈
4. **容错设计** - 重要操作二次确认
